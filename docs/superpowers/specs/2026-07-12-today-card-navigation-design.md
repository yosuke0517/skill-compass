# Today Card Navigation Design

## Goal

Todayを縦長の問題一覧から、1問ずつ集中して解くカード式UIへ変更する。スマホでの片手操作を優先し、左右スワイプと明示的な前後ボタンの両方で問題を移動できるようにする。

## User Experience

- 画面には現在の問題カードを1枚だけ表示する。
- `2 / 5` のような現在位置と回答済み数を表示する。
- 未回答でも前後の問題へ移動できる。
- 未回答の問題はナビゲーション上で明示する。
- 前後ボタンは画面下部に配置し、固定フッターと重ならない。
- 左スワイプで次の問題、右スワイプで前の問題へ移動する。
- 回答送信後は現在カードを回答済み状態に更新し、次の未回答問題へ進みやすくする。
- 最後の問題まで移動した後、未回答が残っていれば未回答問題へ戻れる導線を表示する。
- 翻訳アイコン、回答フォーム、エージェントボタンは現在カードだけに表示する。
- `Add 5` は現在のカード操作を妨げない位置に置き、全問回答後に利用できる。

## Component Boundary

`TodayPage`は問題データ、翻訳データ、回答済み状態を取得する。新しいClient Componentが現在のカード番号を管理し、カード表示とナビゲーションを担当する。

既存の`QuizQuestionCard`は問題1枚の表示・回答送信・翻訳処理を担当する。カード切り替えでコンポーネントが再利用されても、回答済みデータはServer Action後の再検証で正しく反映する。

## Navigation Rules

- 開始位置は最初の問題。
- 前へボタンは最初の問題で無効化する。
- 次へボタンは最後の問題で無効化するのではなく、最後の問題で未回答があれば未回答一覧へ戻る導線を表示する。
- スワイプは横方向の移動量が一定値を超えた場合だけ確定する。
- 縦スクロール中の誤操作を避けるため、横方向の意図が確認できた場合のみスワイプを処理する。
- 回答済みカードへ戻った場合は、回答内容と正解表示を確認できる。

## State and Accessibility

- 現在位置はClient Componentのローカル状態で管理する。
- 現在カードには`aria-current`相当の状態を付与する。
- 前後ボタンにはラベルとdisabled状態を設定する。
- スワイプだけに依存せず、ボタン操作とキーボード操作でも移動できる。
- 遷移時にフォーカスが画面外へ飛ばないよう、カード見出しまたはカードコンテナへフォーカスを移す。

## Error Handling

- 回答送信に失敗した場合、現在カードにエラーを表示し、移動状態は維持する。
- 翻訳やエージェント取得の失敗はカード全体の移動を妨げない。
- 未回答のまま移動しても、未回答状態を失わない。

## Testing

- 初期表示が1枚だけであること。
- 前後ボタンでカードが切り替わること。
- 未回答でも移動できること。
- 最初・最後でボタン状態が正しいこと。
- スワイプで前後移動できること。
- 回答済みカードへ戻ると回答レビューが表示されること。
- 翻訳とエージェント操作が現在カードで動作すること。
- スマホ幅で固定フッターと操作ボタンが重ならないこと。
