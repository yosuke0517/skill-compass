Podcast Studio

信頼できるSources、最新ニュース、任意の個人feed、今日の予定から作る、 非公開の2人会話形式エンジニアリングブリーフィング。

ステータス: 提案段階 モバイルファースト PC管理画面 公開可能な設計

プロダクトループ

情報を、散歩中に聴けるブリーフィングへ

情報収集の頻度とエピソード生成の頻度を分離する。ユーザーが指定した時刻に、 対象となる入力を1本の簡潔な日本語ブリーフィングへまとめる。adminは任意で 英語版も追加生成できる。

01

信頼できるSources、ニュース、任意のCalendar予定、明示的に有効化したSNS情報。

02

根拠付け

一次情報を事実の基準とし、ニュースとSNS投稿は出典付きの背景情報として扱う。

03

生成

テキストmodelが会話を作り、独立した2人話者TTS Providerが音声化する。

04

再生

Skill Compassで再生、download、または非公開配信のbetaを利用する。

永続workflow

MySQL-backed jobで、段階ごとに再開

Web requestはjobをenqueueして応答する。workerはlease付きjobをclaimし、完了した 段階を保存して、失敗地点から再開する。queueは意図的に小さく観測可能に保ち、 将来managed serviceへ差し替えられる。

Queued手動またはscheduled trigger

→

失敗の分離: TTSが失敗しても、収集済みsnapshotと台本は残る。 音声合成だけを再試行し、idempotency keyで同日の重複episodeを防ぐ。

アクセス制御

roleは「誰か」、entitlementは「何ができるか」

商用planと管理権限を分離する。product tierごとにroleを増やさず、plan defaultを user単位で上書きできる。

Free

  • 指定sampleの表示と再生
  • 学習productの体験
  • 非公開生成は利用不可

Pro

  • briefingの生成とdownload
  • Calendar連携
  • 個人X Sourceのopt-in

Admin

  • planと個別overrideの管理
  • 英語版の生成
  • X audiogramの確認と投稿

差し替え可能な境界

Provider選択をproduct logicから分離

episodeの状態遷移は小さなinterfaceだけに依存する。model名、storage vendor、 queue infrastructureは設定で選択可能にする。

Delivery

外部公開より先に、非公開の価値を作る

各releaseを単独で有用に保ちながら、riskの高い外部連携は非公開生成loopが 観測可能かつ安定するまで後ろへ置く。

1. Access

role、plan、entitlement、PC管理画面。

2. Core

手動job、台本、音声、再生、download。

3. Schedule

Source取得間隔、自動briefing、Calendar。

4. Personal

予算上限付きPro timelineとbookmark。

5. Background

player強化と認証付きRSS beta。

6. Publish

英語版、audiogram、確認、X投稿。

7. Operate

保持、legal hold、break-glass、audit。
