Product loop

From signal to a walk-ready briefing

The experience separates collection cadence from episode cadence. At the user's chosen time, all due inputs become one concise Japanese briefing. English is an optional administrator-generated edition.

Durable workflow

MySQL-backed jobs, resumable by stage

Web requests enqueue work and return. A worker claims a leased job, persists each completed stage, and resumes from the failed boundary. The queue is deliberately small, observable, and replaceable by a managed service later.

Connection model

OAuth without password sharing

Users approve access on the provider's own screen. Read scopes and publishing scopes are separate consent steps. Tokens are encrypted at rest and never displayed again.

Access control

Roles answer who. Entitlements answer what.

Commercial plans and administrative authority are independent. Plan defaults can be overridden per user without inventing a new role for every product tier.

Delivery

Build the private value before public distribution

The sequence keeps each release useful while delaying the riskiest integrations until the private generation loop is observable and stable.

1. Access

Roles, plans, entitlements, desktop administration.

2. Core

Manual jobs, scripts, audio, playback, download.

3. Schedule

Source cadence, automatic briefings, Calendar.

4. Personal

Pro timeline and bookmarks with cost limits.

5. Background

Player hardening and authenticated RSS beta.

6. Publish

English edition, audiogram, review, X posting.

7. Operate

Retention, legal hold, break-glass, audit review.
